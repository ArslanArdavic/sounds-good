Architecture Summary

Controllers (3)

FastAPI route handlers for authentication, playlist generation, and user management

Services (6)

Business logic layer: Spotify integration, LLM communication, RAG-based track retrieval, and playlist generation orchestration

Repositories (4)

Data access layer using SQLAlchemy ORM for Users, Tracks, Playlists, and encrypted Spotify tokens

Models (5)

SQLAlchemy entities stored in SQLite with 24-hour cache expiration for track metadata

Utilities (4)

Helpers for prompt construction, track validation, duration matching (±15 min tolerance), and token encryption

Clients (3)

External API wrappers: Spotify Web API, Llama 3.1 70B via Groq, ChromaDB for vector search

RAG Pipeline: User query → EmbeddingService (sentence-transformers) → VectorSearchService → ChromaDB retrieves ~500-1000 relevant tracks → PromptBuilder formats for LLM → LLMService (Llama 3.1 70B) → JSON track IDs → TrackValidator ensures all exist in library → retry with feedback if invalid
