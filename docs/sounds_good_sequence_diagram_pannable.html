Sequence Flow Summary

Step 1: Authentication (Steps 1-22)

Marcus authenticates via Spotify OAuth. System exchanges authorization code for tokens, encrypts and stores refresh token, creates user record.

Step 2: Library Sync (Steps 23-41)

System fetches all 50 playlists and 5,000 tracks from Spotify API, retrieves audio features, stores tracks in SQLite, generates embeddings using sentence-transformers, and indexes in ChromaDB for vector search.

Step 3-4: Input Processing (Steps 42-45)

Marcus enters natural language request, UI initiates playlist generation through PlaylistController.

Step 5a: RAG Retrieval (Steps 46-51)

System encodes user query into vector, performs similarity search in ChromaDB, retrieves top 500-1000 most relevant tracks from user's library based on semantic similarity.

Step 5b: LLM Generation (Steps 52-57)

PromptBuilder formats retrieved tracks into prompt. LLMService sends to Groq's Llama 3.1 70B. LLM analyzes tracks, selects 12 best matches ensuring ~45 min duration, returns JSON with track IDs.

Step 6: Validation & Display (Steps 58-67)

TrackValidator confirms all track IDs exist in library. DurationMatcher checks total duration is within ±15 min tolerance. PlaylistRepository creates playlist record. UI displays results to Marcus.

Step 7: Save to Spotify (Steps 68-77)

Marcus approves playlist. System creates new playlist in Spotify via API, adds all 12 tracks, confirms success to user.
